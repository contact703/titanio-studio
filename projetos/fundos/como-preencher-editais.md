# Guia Completo: Como Preencher e se Inscrever em Editais de Fomento

> **Última atualização:** Março 2026  
> **Público:** Empresas, startups, pesquisadores, projetos culturais e audiovisuais  
> **Foco:** Editais brasileiros (FINEP, BNDES, FAPs, SEBRAE) e internacionais (YC, BID Lab, Banco Mundial)

## PARTE 1 — EDITAIS BRASILEIROS

---

### 1.1 — Como Ler um Edital: Estrutura e Termos Técnicos

#### Seções que você DEVE ler primeiro (nessa ordem):

1. **OBJETO** — O que o edital financia. Ex: "O presente edital tem por objeto apoiar projetos de inovação tecnológica..."  
   ➜ Se seu projeto não se encaixa no objeto, pare aqui.

2. **BENEFICIÁRIOS ELEGÍVEIS** — Quem pode se inscrever. Verifique:
   - Tipo jurídico (MEI, ME, EPP, Média, Grande)
   - Tempo de constituição exigido
   - CNAE permitido (código de atividade econômica)
   - Localização geográfica (nacional, estadual, municipal)

3. **DESPESAS ELEGÍVEIS** — O que pode ser pago com os recursos:
   - **Custeio:** Materiais, serviços, viagens, bolsas
   - **Capital:** Equipamentos, infraestrutura
   - **Vedações:** O que NÃO pode (ex: pagamento de dívidas, salários de sócios)

4. **CRITÉRIOS DE AVALIAÇÃO** — Como sua proposta será pontuada. Escreva para maximizar esses critérios.

5. **CRONOGRAMA DO EDITAL** — Datas de abertura, submissão, resultado, contratação.

6. **FORMA DE APOIO** — Reembolsável (empréstimo) vs Não-reembolsável (subvenção/grant) vs Participação acionária.

#### Glossário Essencial:

| Termo | Significado Prático |
|-------|---------------------|
| **Subvenção Econômica** | Dinheiro que não precisa devolver |
| **Reembolsável** | Empréstimo, geralmente com juros baixos |
| **Contrapartida** | Valor que você precisa aportar (financeira ou econômica) |
| **Enquadramento** | Verificação se você atende todos os requisitos |
| **Fase de Diligência** | Análise documental antes da aprovação final |
| **Vigência** | Período de execução do projeto |
| **Prestação de Contas** | Relatórios e comprovantes de gastos |
| **Interveniente** | Parceiro da proposta (ICT, universidade, etc.) |
| **ICT** | Instituição Científica e Tecnológica (universidades, Embrapa, etc.) |
| **TRL** | Technology Readiness Level — grau de maturidade tecnológica (1-9) |

#### ⚠️ Checklist de Leitura Rápida (30 minutos):

- [ ] Qual é o objeto exato?
- [ ] Minha empresa é elegível? (CNPJ ativo, CNAE correto, porte certo)
- [ ] Qual o valor máximo financiável?
- [ ] É reembolsável ou não-reembolsável?
- [ ] Qual a contrapartida exigida? (geralmente 10-30% do projeto)
- [ ] Quais documentos são necessários?
- [ ] Qual a data limite de submissão?
- [ ] Como e onde submeter? (plataforma eletrônica, físico?)
- [ ] Há exigência de parceria com universidade?

### 1.2 — Plano de Trabalho: FINEP / CNPq / FAPs

O Plano de Trabalho é o coração da proposta. Estrutura típica:

#### Seções Obrigatórias (FINEP Subvenção Econômica):

**1. CARACTERIZAÇÃO DA EMPRESA**
- Histórico, expertise, diferenciais
- Equipe técnica (curriculum vitae dos responsáveis)
- Capacidade instalada

**2. PROBLEMA / OPORTUNIDADE**
- Qual problema de mercado você resolve?
- Dados que comprovem a relevância (estatísticas, pesquisas de mercado)
- Por que agora? (janela de oportunidade)

**3. DESCRIÇÃO DO PRODUTO/SERVIÇO/PROCESSO**
- O que será desenvolvido
- Diferenciais tecnológicos (novidade técnica)
- TRL atual e TRL ao final do projeto
- Propriedade intelectual (tem patente? vai registrar?)

**4. OBJETIVOS**

Escreva objetivos com o método **SMART:**
- **S**pecífico — "Desenvolver algoritmo de IA para triagem de laudos médicos"
- **M**ensurável — "com acurácia de 92%"
- **A**tingível — baseado na sua capacidade real
- **R**elevante — alinhado ao objeto do edital
- **T**emporal — "em 18 meses"

> **Exemplo ruim:** "Criar uma solução inovadora para o setor de saúde"  
> **Exemplo bom:** "Desenvolver e validar um módulo de triagem automatizada de laudos radiológicos baseado em deep learning, atingindo sensibilidade ≥92% e especificidade ≥88% em banco de dados clínico com mínimo de 10.000 imagens, em 18 meses de execução."

**5. METODOLOGIA**
- Descreva como vai executar cada etapa
- Referencie métodos científicos/tecnológicos reconhecidos
- Mostre que a equipe tem competência para executar
- Identifique riscos e como vai mitigá-los

**6. METAS E INDICADORES**

Formato de tabela obrigatório em muitos editais:

| Meta | Descrição | Indicador | Valor esperado | Prazo |
|------|-----------|-----------|----------------|-------|
| 1 | Protótipo funcional do módulo de IA | Versão beta entregue | 1 protótipo | Mês 6 |
| 2 | Validação clínica | Relatório técnico aprovado | Acurácia ≥92% | Mês 12 |
| 3 | Publicação científica | Artigo submetido a periódico Qualis A2 | 1 artigo | Mês 18 |

**7. CRONOGRAMA**

Divida em trimestres ou semestres. Use formato de Gantt simplificado:

| Atividade | M1-3 | M4-6 | M7-9 | M10-12 | M13-15 | M16-18 |
|-----------|------|------|------|--------|--------|--------|
| Revisão bibliográfica | ✓ | | | | | |
| Desenvolvimento do algoritmo | ✓ | ✓ | ✓ | | | |
| Testes internos | | | ✓ | ✓ | | |
| Validação clínica | | | | ✓ | ✓ | |
| Relatório final | | | | | | ✓ |

**8. ORÇAMENTO**

#### Como Calcular o Orçamento:

**a) Recursos Humanos (maior parte do custeio):**
```
Pesquisador Sênior: R$ 12.000/mês × 18 meses × 40% dedicação = R$ 86.400
Desenvolvedor Pleno: R$ 8.000/mês × 18 meses × 80% dedicação = R$ 115.200
Bolsista de Iniciação Tecnológica: R$ 1.500/mês × 18 meses × 100% = R$ 27.000
TOTAL RH: R$ 228.600
```

**b) Serviços de Terceiros:**
```
Empresa de validação clínica (contrato): R$ 45.000
Consultoria de propriedade intelectual: R$ 8.000
Auditoria técnica independente: R$ 12.000
TOTAL SERVIÇOS: R$ 65.000
```

**c) Material de Consumo:**
```
Licenças de software: R$ 15.000
Material de laboratório: R$ 8.000
TOTAL MATERIAL: R$ 23.000
```

**d) Capital (Equipamentos):**
```
Servidor GPU para treinamento de IA: R$ 85.000
TOTAL CAPITAL: R$ 85.000
```

**TOTAL DO PROJETO: R$ 401.600**
**Solicitado ao Edital (80%): R$ 321.280**
**Contrapartida da empresa (20%): R$ 80.320**

#### ⚠️ Erros que Eliminam a Proposta:

1. **Não ler o edital completamente** — elimina por descumprimento de regras
2. **CNPJ com pendências** (Receita Federal, FGTS, INSS) — bloqueio na diligência
3. **Objeto incompatível** — propor produto que não se encaixa no tema do edital
4. **Metas sem indicadores quantitativos** — "melhorar a qualidade" não é meta
5. **Orçamento superfaturado** (>30% acima do mercado) — suspeita e corte
6. **Copiar seção de outro projeto** — avaliadores identificam textos genéricos
7. **Equipe sem qualificação comprovada** — currículo não evidencia capacidade
8. **Não mencionar inovação** em editais de inovação — óbvio, mas acontece
9. **Contrapartida abaixo do mínimo** — inabilitação automática
10. **Submissão no último dia** — sistemas travam; envie 48h antes

### 1.4 — Estratégias de Escrita para Editais Brasileiros

#### Como Comunicar Impacto Social e Econômico:

Avaliadores precisam justificar internamente a aprovação. Dê a eles argumentos:

**Framework dos 3 impactos:**
1. **Impacto econômico:** Geração de empregos, aumento de faturamento, redução de custos
2. **Impacto social:** Acesso, inclusão, saúde, educação, qualidade de vida
3. **Impacto para o ecossistema:** Publicações, patentes, formação de pesquisadores, spin-offs

**Exemplo de parágrafo de impacto:**
> "O presente projeto tem potencial para gerar 12 empregos diretos qualificados ao longo de 3 anos, contribuindo para a formação de um polo de inteligência artificial na região Norte Fluminense. A solução desenvolvida poderá reduzir em 40% o tempo de diagnóstico radiológico em hospitais públicos, impactando diretamente uma população de 180.000 pessoas atendidas pelo SUS no município de Maricá-RJ. Estima-se ainda a possibilidade de licenciamento da tecnologia para 15 municípios vizinhos nos 24 meses posteriores ao projeto."

#### Linguagem Técnica vs Linguagem de Negócios:

| Não escreva assim | Escreva assim |
|---|---|
| "Vamos desenvolver uma IA muito boa" | "Desenvolveremos um modelo de deep learning (CNN ResNet-50) treinado com dataset proprietário de 50.000 imagens" |
| "O mercado é grande" | "O mercado endereçável de teleradiologia no Brasil é estimado em R$ 2,3 bilhões (IMARC, 2025)" |
| "Nossa equipe é experiente" | "A equipe é composta por 1 doutor em Ciência da Computação com 12 anos de experiência em visão computacional e 2 publicações em periódicos Qualis A1" |
| "O projeto é inovador" | "A solução possui diferencial tecnológico patenteável, com pedido de patente depositado no INPI sob nº BR102025000001-0" |

#### Como se Diferenciar da Concorrência:

1. **Cite e supere o estado da arte** — "As soluções existentes (X, Y) atingem 85% de acurácia. Nossa abordagem alcança 92%."
2. **Mostre barreiras de entrada** — dados proprietários, parcerias exclusivas, know-how de equipe
3. **Evidencie tração** — clientes pilotos, cartas de intenção, PoCs realizadas
4. **Demonstre por que agora** — mudança regulatória, tecnologia que amadureceu, problema urgente

### 2.1 — Diferenças Culturais no Preenchimento

#### Estilo Americano (aceleradoras, VCs, SBIR/STTR):
- **Direto ao ponto:** Chegue na solução em 2 parágrafos
- **Foco em mercado:** Sempre mostrar TAM/SAM/SOM
- **Tração é rei:** Mostre números reais, mesmo que pequenos
- **"Why now":** Explique por que o timing é perfeito
- **Founder-market fit:** Por que VOCÊ é a pessoa certa para resolver isso?
- Evite: prolixidade, disclaimers excessivos, linguagem burocrática

#### Estilo Europeu (Horizon Europe, EIC, EIF):
- Mais técnico e detalhado que o americano
- Forte ênfase em impacto social e ambiental (SDGs, Green Deal)
- Exige parceria com instituições de pelo menos 3 países europeus
- Valoriza publicações, propriedade intelectual, patentes
- Importante: disseminação dos resultados para a comunidade científica

#### Estilo Multilateral (BID, Banco Mundial, PNUD):
- **Linguagem de desenvolvimento:** impacto, inclusão, vulnerabilidade
- Obrigatório: alinhamento com ODS (Objetivos de Desenvolvimento Sustentável)
- Mensuração de impacto social é central (não apenas financeiro)
- Favorecem soluções que beneficiem populações de baixa renda
- Teoria da Mudança (Theory of Change) é documento separado

#### Como Adaptar para Investidor Americano:

**O que funciona no Brasil mas não nos EUA:**
- ❌ Texto longo sobre histórico da empresa antes do problema
- ❌ Linguagem formal ("Vem por meio desta...")
- ❌ Foco excessivo em tecnologia antes de mostrar mercado
- ❌ Números sem fonte clara

**O que funciona nos EUA:**
- ✅ Começar com o problema de forma visceral ("1 em cada 3 diagnósticos...")
- ✅ Mostrar tração cedo ("R$200k em LOIs assinadas")
- ✅ Equipe concisa com 1-2 linhas por pessoa
- ✅ Ask claro: "Pedimos US$500k para alcançar X milestone"

#### Estrutura Problem Statement / Solution / Market Size:

**Problem Statement:**
> "[Dado numérico] de [público] sofrem com [problema específico e mensurável]. As soluções atuais [X, Y, Z] falham porque [razão específica], resultando em [consequência negativa quantificada]."

**Solution:**
> "[Nome da solução] é um [categoria de produto] que [mecanismo único] para [público-alvo], resultando em [benefício mensurável]. Diferente de [concorrente], fazemos [diferencial específico]."

**Market Size (TAM/SAM/SOM):**
> - **TAM:** US$5B — mercado global de teleradiologia
> - **SAM:** US$800M — América Latina (hospitais com >50 leitos)
> - **SOM:** US$40M — Brasil nos primeiros 3 anos (meta de 5% do SAM)

### 2.2 — Y Combinator Application

#### As Perguntas Mais Importantes do YC:

**"What are you making?"** (responda em 1 frase, 7 palavras ou menos)
> ❌ "Desenvolvemos uma plataforma de inteligência artificial para diagnóstico médico que utiliza..."  
> ✅ "AI that reads X-rays faster than radiologists."

**"Describe what your company does in 50 characters"**
> "AI radiology assistant for Latin America"

**Perguntas típicas e o que o YC realmente quer saber:**

| Pergunta | O que realmente querem | Armadilha a evitar |
|----------|------------------------|-------------------|
| "Why did you pick this idea?" | Founder-market fit | Dizer "vi uma oportunidade de mercado" |
| "How far along are you?" | Tração real | Inflacionar números |
| "Why is now the right time?" | Timing único | Respostas genéricas |
| "What do you understand about your problem that others don't?" | Insight não-óbvio | Insights óbvios ("o mercado é grande") |
| "Who are your competitors?" | Honestidade + posicionamento | Dizer "não temos concorrentes" |

#### Dicas Específicas para Founders Brasileiros:

1. **Traduza métricas para USD** — mesmo que o negócio seja em R$, apresente em dólares
2. **Contextualize o mercado brasileiro** — avaliadores americanos podem não saber o tamanho do Brasil
3. **Mostre por que o Brasil é vantajoso** — pool de engenheiros talentosos e baratos, mercado de 215M pessoas pouco explorado por tech
4. **Endereça o "por que não Silicon Valley faz isso?"** — barreiras locais, regulação, idioma
5. **Se tiver usuários, mostre retenção** — YC adora DAU/MAU, churn, NPS
6. **Não mencione que é startup de impacto para o YC** — foque no crescimento, não no impacto (o impacto é consequência)
7. **Video de 1 minuto é obrigatório** — natural, câmera frontal está ok, sem roteiro decorado

#### Checklist YC Application:
- [ ] Consegue explicar em 1 frase o que faz?
- [ ] Tem tração para mostrar? (mesmo que pequena, mostre)
- [ ] A equipe tem ao menos 1 técnico forte (engenheiro)?
- [ ] Tem pelo menos 2 fundadores? (preferência do YC)
- [ ] Por que você? Por que agora? Por que esse mercado?
- [ ] Você conversou com pelo menos 50 clientes potenciais?
- [ ] Qual é o plano dos primeiros 3 meses se entrar no YC?

### 2.3 — BID Lab / Banco Mundial

#### Linguagem de Desenvolvimento:

Termos-chave que DEVEM aparecer na proposta:

- **ODS (SDGs):** Sempre citar quais Objetivos de Desenvolvimento Sustentável são endereçados
- **Populações vulneráveis / BOP (Base of Pyramid):** Famílias de baixa renda, mulheres, jovens, comunidades rurais
- **Inclusão financeira / digital / social**
- **Escalabilidade:** "O modelo pode ser replicado em outros países da América Latina"
- **Sustentabilidade:** O projeto continua depois que o financiamento acaba?
- **Additionality:** O projeto só acontece COM esse financiamento?

#### Como Mensurar Impacto Social:

**Framework IRIS+ (padrão do BID/Banco Mundial):**

| Dimensão | Indicador | Como Medir |
|----------|-----------|------------|
| Alcance | Nº de pessoas beneficiadas diretamente | Cadastros, transações |
| Profundidade | Variação de renda / acesso / qualidade de vida | Pesquisa de campo (linha de base + endline) |
| Sustentabilidade | % que continuou usando após 6 meses | Dados de retenção |
| Gênero | % de mulheres beneficiadas | Desagregação por gênero nos dados |
| Localização | % em áreas vulneráveis (IBGE) | Cruzamento com dados do IBGE |

**Exemplo de Impact Metrics para o BID:**
```
- 15.000 famílias de baixa renda com acesso a diagnóstico radiológico 
  (hoje, apenas 23% das cidades com menos de 50k hab. têm radiologistas)
- Redução de 35% no tempo médio de espera por diagnóstico (de 45 para 29 dias)
- 60% dos beneficiados em municípios com IDH abaixo de 0,700
- 52% de mulheres entre os beneficiados
- ODS alinhados: ODS 3 (Saúde), ODS 10 (Redução das Desigualdades), ODS 11 (Cidades)
```

#### O que é Theory of Change (ToC):

A Teoria da Mudança explica a lógica de como suas atividades geram impacto.

**Estrutura:**
```
INSUMOS → ATIVIDADES → PRODUTOS → RESULTADOS → IMPACTO

Insumos: 
  - R$500k do BID Lab
  - Equipe de 8 pessoas
  - Parceria com 3 hospitais públicos

Atividades:
  - Desenvolvimento do software de IA
  - Treinamento de técnicos hospitalares
  - Integração com sistemas SUS

Produtos (Outputs):
  - 1 software de IA implantado
  - 50 técnicos treinados
  - 3 hospitais integrados

Resultados (Outcomes - curto/médio prazo):
  - Tempo de diagnóstico reduzido em 35%
  - 15.000 pacientes atendidos no ano 1
  - Satisfação dos técnicos: 85% positivo

Impacto (longo prazo):
  - Redução da mortalidade por diagnóstico tardio em regiões sem radiologistas
  - Modelo replicado para 20 municípios até 2028
```

### 3.2 — Consultores: Vale o Investimento?

#### Quando contratar um consultor:

✅ **Vale a pena quando:**
- O edital tem mais de R$ 500k em jogo
- Sua empresa nunca submeteu uma proposta
- O edital exige parceria com ICT (universidade)
- Você não tem equipe interna com tempo disponível
- O edital tem exigências técnicas complexas (ABNT, normas específicas)

❌ **Não vale quando:**
- O consultor pede percentual sobre o valor aprovado (pode ser antiético e ilegal em alguns editais)
- O valor solicitado é pequeno (<R$100k)
- Você já tem experiência e equipe interna
- O "consultor" não tem referências comprovadas

#### Como Avaliar um Consultor:

**Perguntas obrigatórias antes de contratar:**
1. Quantas propostas submeteu nos últimos 2 anos? Taxa de aprovação?
2. Tem experiência específica neste edital (FINEP? BID Lab?)
3. Pode fornecer 3 referências de clientes aprovados?
4. Como é a cobrança? (por projeto, por hora, percentual?)
5. Quem efetivamente escreve? (ele ou estagiário?)

**Modelo de cobrança saudável:**
- Diagnóstico inicial: R$ 1.500 – R$ 5.000 (único)
- Elaboração da proposta: R$ 8.000 – R$ 30.000 (dependendo da complexidade)
- Acompanhamento/diligência: R$ 3.000 – R$ 8.000
- ❌ Evitar: "só cobro se aprovar X% do valor" (conflito de interesse, pode inflar orçamento)

### 3.3 — Como Usar IA para Editais

#### O que a IA faz bem:

1. **Estruturar o Plano de Trabalho** — Descreva seu projeto e peça para estruturar em seções
2. **Revisar linguagem** — "Reescreva esse parágrafo de forma mais técnica para avaliador de P&D"
3. **Calcular orçamento** — "Monte uma planilha de custos para um projeto de 18 meses com X profissionais"
4. **Traduzir e adaptar** — "Adapte esse texto para o estilo YC Application em inglês"
5. **Encontrar indicadores** — "Que indicadores de impacto posso usar para uma solução de saúde digital para o BID?"
6. **Revisar aderência ao edital** — Cole o edital e sua proposta e peça análise de gaps
7. **Simular perguntas de avaliador** — "Que objeções um avaliador teria sobre essa proposta?"

#### Prompt Templates Úteis:

**Para estruturar proposta:**
```
Estou escrevendo uma proposta para o edital [NOME DO EDITAL].
O objeto do edital é: [COLE O OBJETO]
Meu projeto é: [DESCREVA EM 5 LINHAS]
Por favor, estruture um Plano de Trabalho completo com: 
problema, solução, objetivos SMART, metodologia, metas com indicadores, 
cronograma de 18 meses e orçamento estimado.
```

**Para revisar aderência:**
```
Aqui está o trecho de critérios de avaliação do edital:
[COLE OS CRITÉRIOS]

E aqui está minha proposta atual:
[COLE SUA PROPOSTA]

Analise: minha proposta atende todos os critérios? 
O que está faltando? O que está fraco? 
Liste por ordem de prioridade de melhoria.
```

**Para adaptar para YC:**
```
Aqui está minha empresa descrita em português:
[COLE DESCRIÇÃO]

Por favor:
1. Reescreva em inglês no estilo YC Application (direto, com números, founder-market fit)
2. Crie uma descrição de 7 palavras ou menos
3. Crie uma descrição de 50 caracteres ou menos
4. Escreva 3 possíveis respostas para "Why you?" em 3 parágrafos
```

#### ⚠️ Limitações da IA em Editais:

- **Não conhece editais específicos em tempo real** — sempre verifique o texto oficial
- **Pode inventar dados e referências** — confirme todas as estatísticas
- **Não substitui revisão humana especializada** — use como rascunho, não como versão final
- **Não preenche sistemas** (Plataforma Carlos Chagas, SAP da FINEP) — só o texto
