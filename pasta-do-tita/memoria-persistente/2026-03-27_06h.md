## 💡 Lições Aprendidas

### 1. **Magic.link Private Keys São Não-Exportáveis (Segurança por Design)**
- **Descoberta:** Tentativas de extrair private key via localStorage, IndexedDB, e Service Workers falharam
- **Por quê:** Magic.link usa secure enclave no servidor — key nunca exposta no browser
- **Workaround:** Playwright + OTP login → salva cookies → opera como usuário autenticado
- **Aplicação futura:** Serviços com MPC (Multi-Party Computation) como Magic.link exigem automação de UI, não de API L1

### 2. **Polymarket L2 Auth Usa urlsafe_b64, Não b64 Standard**
- **Erro inicial:** Assinatura HMAC rejeitada (401 Unauthorized)
- **Fix:** Trocar `base64.b64encode` por `base64.urlsafe_b64encode` (padding `=` vira `-` e `_`)
- **Detalhe crítico:** Address correto pra auth é **signer** (`0x5a9A...`), não proxy wallet
- **Lição:** Sempre verificar encoding exact (standard vs urlsafe vs no-padding) em APIs de criptomoedas

### 3. **Rate Limiting Severo da Anthropic → Necessidade de Fallback Model**
- **Evento:** Claude Opus 4 retornou 429 (rate limit exceeded) 4x seguidas às 18:42 BRT
- **Impacto:** Heartbeat falhou completamente, sem resposta
- **Solução proposta:** Configurar fallback automático pra modelo mais barato (Haiku, Kimi) quando rate limit detectado
