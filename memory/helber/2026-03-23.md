## Apple App Store — 2ª Rejeição Manda a Nota iOS

### Problema
- App rejeitado pela **2ª vez** por **Guideline 3.1.1** (In-App Purchase obrigatório)
- Review date: 21/03/2026
- Device: iPhone 17 Pro Max
- Submission ID: ff1f290d-ca54-4d0c-adfc-a0cb0eb0f6e0
- Motivo: "subscriptions can be purchased in the app using payment mechanisms other than In-App Purchase"

### Causa Raiz Identificada
O `PricingModal.tsx` já usava StoreKit/IAP corretamente, MAS **3 pontos no código** ainda abriam o site externo com Stripe:

| Arquivo | Linha | Problema |
|---|---|---|
| `profile.tsx` | 232 | Botão "⚡ Fazer Upgrade" → `Linking.openURL('https://mandaanota.titanio.studio/planos')` |
| `profile.tsx` | 242 | Botão "Ver Planos" → `Linking.openURL('https://mandaanota.titanio.studio/planos')` |
| `chat.tsx` | 473 | Banner de limite atingido → `Linking.openURL('https://mandaanota.titanio.studio/planos')` |
| `tutorial.tsx` | 23 | Texto mencionava "Acesse mandaanota.titanio.studio/planos" |

Quando o usuário clicava nesses botões → redirecionava para `checkout.stripe.com` (pagamento externo) → violação Guideline 3.1.1.

### Correções Aplicadas
1. `profile.tsx` L232: `Linking.openURL(...)` → `router.push('/PricingModal')`
2. `profile.tsx` L242: `Linking.openURL(...)` → `router.push('/PricingModal')`
3. `chat.tsx` L473: `Linking.openURL(...)` → `router.push('/PricingModal')`
4. `chat.tsx` L1: Removido `Linking` do import (não mais usado)
5. `tutorial.tsx` L23: Texto alterado para "Acesse a aba 'Planos' dentro do app"
