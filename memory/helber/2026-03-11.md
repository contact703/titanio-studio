### 🛡️ Resiliência — Keep-Alive + Retry

**Keep-Alive:**
- Manda a Nota pinga Shadow API cada 4 minutos
- Endpoint `/health` e `/ping` adicionados na Shadow API
- Logs: `[KEEP-ALIVE] Shadow API ping OK`

**Retry com Backoff:**
- 3 tentativas em erros de conexão (fetch failed, timeout)
- Backoff exponencial: 2s → 3s → 4.5s
- NUNCA retry se recebeu resposta do servidor (evita duplicação)

### 💰 NFS-e Automática de Vendas

**Problema:** Tabela `billing_orders` não existia, nota não era emitida
**Solução:** Migração criada + fluxo completo implementado

**Fluxo:**
1. Usuário compra plano no Stripe
2. Webhook `checkout.session.completed` chamado
3. Registro em `nfse_billing_orders`
4. Plano do usuário atualizado
5. NFS-e emitida automaticamente (Titânio → Comprador)
6. PDF baixado e enviado por email (tomador + prestador)

**Variáveis de Ambiente:**
```
NFSE_CERT_PFX_BASE64 = certificado A1 da Titânio
NFSE_CERT_PASSPHRASE = senha do certificado
NFSE_PRESTADOR_CNPJ = 08103457000133
NFSE_PRESTADOR_IM = 11110280011
NFSE_PRESTADOR_MUN_IBGE = 3106200 (BH)
NFSE_PRESTADOR_SIMPLES = optante
NFSE_PRESTADOR_EMAIL = processos@portoegomes.com.br
ADMIN_SECRET_KEY = (chave para endpoint admin)
```

**Descrição da Nota:**
`Compra de plano de aplicativo - Plano [Nome]`

**Valor:** Direto do Stripe (`session.amount_total`)

**Endpoint Admin (reemissão manual):**
```
POST /api/admin/emit-sale-nfse
Header: X-Admin-Key: <ADMIN_SECRET_KEY>
Body: { userId, planType, amountCents }
```
