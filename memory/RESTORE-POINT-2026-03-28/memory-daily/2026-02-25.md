## Authorization Form Fix — ✅ RESOLVED
Pedro complained "Authorization Form" button appeared on every page.

### Root cause
The button was injected by **JavaScript inside HTML widgets** in THREE places:
1. **Template #1502** (Elementor Single Page) — had 2 HTML widgets injecting the script
2. **Page #1788** (Locations) — had 1 HTML widget (id: 80d8967)
3. **Footer template #188** — had **5 HTML widgets** (ids: 1c11aec, 0db5e6f, bdbe1c1, 66c1ca8, d5c55c1)

The script was clever — it checked for `.elementor-1502` and `[data-id="8054783"]` (Post Content widget), then dynamically created a styled button linking to `/filming-authorization-form/`. It also handled PT/EN localization.

### Fix
Deleted all HTML widgets from template #1502, page #1788, and footer #188 via Elementor JS API:
- `$e.run('document/elements/delete', {container: widget})`
- `elementor.saver.update()` to publish

## Technical Lessons Learned
1. **Elementor JS API is the way to go** — clicking UI buttons via browser automation is unreliable. Use:
   - `elementor.documents.getCurrent().container` to navigate widget tree
   - `$e.run('document/elements/delete', {container})` to delete
   - `$e.run('document/elements/settings', {container, settings})` to modify
   - `elementor.saver.update()` to save/publish
2. **Subagents struggle with Elementor** — they timeout, hit rate limits, or get stuck on login. Better to do complex Elementor work myself
3. **Use Kimi K2.5 for subagents** — avoids Claude 429 rate limits
4. **HTML widget content not readable via `.get('settings').html`** — returned empty strings even though widgets had content. But delete still works
5. **Footer template ID is 188** — this is where site-wide scripts live
6. **Page structure**: Header(154) → Single Page Template(1502) → Page Content(post ID) → Footer(188)
