## Problema do Site Maricá Film Commission - RESOLVIDO ✅

### O que aconteceu
Eduardo relatou que o site estava travando - não navegava, não mudava de idioma, travava com meia página carregada.

### Causa raiz
**Mixed Content** - As fontes locais do Google (Elementor) estavam com URLs `http://`:
- `http://maricafilmcommission.com/wp-content/uploads/elementor/google-fonts/css/dmsans.css`
- `http://maricafilmcommission.com/wp-content/uploads/elementor/google-fonts/css/worksans.css`

Quando o navegador tenta carregar recursos HTTP em uma página HTTPS, ele **bloqueia** por segurança. As fontes não carregavam → o site ficava esperando → travava.

### Como descobri
1. Playwright reportou: `Mixed Content: requested an insecure stylesheet`
2. curl mostrou os CSS com URLs HTTP internas
3. O Elementor salvou as fontes quando o Site URL ainda era HTTP

### Solução aplicada
1. **Replace URL do Elementor**: Troquei todas URLs de `http://` para `https://` (48 linhas no banco de dados)
2. **Limpei cache do Elementor**
3. O Elementor regenerou os arquivos CSS com HTTPS

### Segunda investigação (14:00-14:25)

Eduardo reportou que o site ainda travava no browser. Investiguei mais:

**Descoberta crítica:** Quando desativei o LiteSpeed Cache, perdi o lazy loading de JavaScript!
- 483KB de scripts estavam bloqueando a renderização
- 0 scripts tinham defer/async
- O LiteSpeed Cache converte scripts para `type="litespeed/javascript"` e carrega depois

**Solução:** Reativei o LiteSpeed Cache
- Agora: 6 scripts lazy loaded, 0 bloqueantes
- Headers de cache presentes
- Fontes continuam HTTPS (Mixed Content corrigido)

**Lição aprendida:** O LiteSpeed Cache não era o problema - era parte da solução! Ele faz lazy loading de JS que é essencial para o site funcionar.
